Confluence Installing

Confluence Installing

环境

  • ubuntu==14.04
  • confluence==5.5
  • postgres==9.4
  • java==1.7

##安装步骤

下载需要的安装包 <需要添加链接>

解压并安装

$ ./atlassian-confluence-5.5.4-x64.bin

打开浏览器，http://localhost:8090

记下这里的Server ID：ADFI-2763-6EJC-35Q4

###破解

#####停止服务
$ service confluence stop

$ unzip confluence5.1-crack.zip

$ cd confluence5.1-crack

$ ls

atlassian-extras-2.4.jar iNViSiBLE
$ ./keygen.sh

###导入破解文件
将破解后的授权文件导入到目录中

$ cp atlassian-extras-2.4.jar /usr/local/confluence/confluence/WEB-INF/lib/

##创建confluence数据库
创建名为confluence的pg数据库和新建一个用于confluence的用户

###重启服务

###web安装

按照安装引导，输入相关字段进行安装

###汉化

导入汉化文件

$cp Confluence-5.4.4-language-pack-zh_CN.jar /opt/atlassian/confluence/confluence/WEB-INF/lib/

汉化完重启
$ service confluence restart

git

添加远程库：

本地创建Git库

在Github上创建一个Git仓库，复制仓库名，例如git@github.com:SeanPC/me.git。

然后在本地执行

git remote add origin git@github.com:SeanPC/me.git

推送本地数据到Git

git push -u origin master

把本地库内容推送到远程，用git push命令，实际上是把当前分支master推送到远程

由于远程库是空的，第一次推送master分支时，加上了-u参数，Git不但会把本地分支内容推送到新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令。

至此,只要本地做了提交，就可以通过命令
git push origin master

把本地master分支的最新修改内容推送到Git上。现在就做到了真正的分布式版本库。

注意：
有些使用https的每次都需要验证，而ssh则不需要。

从远程库克隆：

在Git上创建一个新的仓库，并勾选Initialize this repository with a README，这样Git就会自动创建一个README.me文件，创建完毕之后就能看到README.me文件

在本地

git clone *

完成！
