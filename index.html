Gitlab Installing

发表于 2017-08-25

#Gitlab 简介
GitHub是2008年由Ruby on Rails编写而成，与业界闻名的Github类似;但要将代码上传到GitHub上面，而且将项目设为私有还要收费。GitLab 是一个用于仓库管理系统的开源项目，使用Git作为代码管理工具，可通过Web界面进行访问公开的或者私人项目，非常适合在团队内部使用。
在gitlab中有三个版本，分别是CE（社区版）、EE（企业版）、OM（RPM包完整版，里面包括nginx、Redis等其它软件，比较大）。这里的编译安装版，是指CE版的源码安装
官网https://docs.gitlab.com/

##使用omnibus packages安装

###在 Ubuntu 14 安装
使用国内安装源镜像，加快安装速度。修改/etc/apt/sources.list.d/gitlab-ce.list:

curl https://packages.gitlab.com/gpg.key 2> /dev/null | sudo apt-key add - &>/dev/null
vi /etc/apt/sources.list.d/gitlab-ce.list
deb https://mirrors.tuna.tsinghua.edu.cn/gitlab-ce/debian jessie main
开始安装
安装依赖包
sudo apt-get install curl openssh-server ca-certificates postfix
安装 GitLab 社区版
apt-get install gitlab-ce
初始化，初始化完自动启动 GitLab
sudo gitlab-ctl reconfigure

###问题

####CXABI_1.3.8….not found
解决方案

sudo add-apt-repository ppa:ubuntu-toolchain-r/test
sudo apt-get update
sudo apt-get install gcc-4.9 g++-4.9

####Excepted process tp exit with [0], but received ‘137’
内存不足，导致OOM关闭，解决方案增加内存。

git

发表于 2017-08-15

添加远程库：

本地创建Git库

在Github上创建一个Git仓库，复制仓库名，例如git@github.com:SeanPC/me.git。

然后在本地执行

git remote add origin git@github.com:SeanPC/me.git

推送本地数据到Git

git push -u origin master

把本地库内容推送到远程，用git push命令，实际上是把当前分支master推送到远程

由于远程库是空的，第一次推送master分支时，加上了-u参数，Git不但会把本地分支内容推送到新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令。

至此,只要本地做了提交，就可以通过命令
git push origin master

把本地master分支的最新修改内容推送到Git上。现在就做到了真正的分布式版本库。

注意：
有些使用https的每次都需要验证，而ssh则不需要。

从远程库克隆：

在Git上创建一个新的仓库，并勾选Initialize this repository with a README，这样Git就会自动创建一个README.me文件，创建完毕之后就能看到README.me文件

在本地

git clone *

完成！
